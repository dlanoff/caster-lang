Caster Language

Script-like readability, native-like power.

Caster is a small compiled experimental language for writing code that feels quick and readable, but still turns into plain C under the hood. You get typed function signatures so APIs are clear, inferred local variables so functions stay light, simple arrays and maps for everyday data work, and explicit references when a function needs to mutate something outside itself.

Overview

Caster is built to make everyday code feel as direct as a scripting language while compiling to C with a clear, predictable runtime model. Function boundaries are explicitly typed, local variables are inferred, and values copy by default. Mutation that can affect the caller is made visible through REF, while function-local borrows remain scoped to the values that own them.

Typed edges

Function parameters and return values are explicit, so APIs stay readable and compiler-checkable.

Inferred bodies

Simple local variables are inferred, while locals that start from user-written function calls name the function return type.

Value-first data

Normal values copy. Mutation that affects the caller is visible through REF .

C as the backend

Caster emits C and links against a small runtime for strings, arrays, maps, JSON, tasks, and HTTP.

Why Caster

Caster tries to keep the parts that make C feel powerful while removing the parts that make small programs noisy. You still see mutation. You still know where references cross function boundaries. You still get generated C. But you also get arrays, maps, strings, JSON, route helpers, typed tasks, and cleanup rules that are part of the language instead of scattered across hand-written C code.

First Program

A function begins with FN , declares its parameter types, and always declares a return type. Bodies use Caster's parenthesized block form. 

FN greet(STR name) -> STR = (
    ret "Hello, " + name
)

FN main() -> NUL = (
    STR message = greet("Caster")
    log(message) // Hello, Caster
)

The stored result of greet(...) is explicitly typed because it crosses a function-call boundary. Values derived locally from message may be inferred.

Program Structure

Imports live first, followed by MAP declarations and functions. Function contracts are collected before bodies are analyzed, so helpers can be written below the code that uses them. The only special placement rule is main() : it can live near the top for readability, and the compiler emits it after helper functions in generated C. 

use WEB

FN main() -> NUL = (
    app = WEB.app({
        port = 8000
    })

    WEB.get(REF app, "/accounts/:id", [auth], account)

    WEB.listen(REF app)
)

FN auth(REF[WEB.Req] req) -> BOL = (
    if(!req.headers.has("authorization"))(
        *req.res = WEB.json({
            error = "missing authorization"
        }, 401)

        ret false
    )

    *req.ctx = req.ctx + {
        "userId" = "user_123"
    }

    ret true
)

FN account(REF[WEB.Req] req) -> WEB.Res = (
    payments = [
        { amount = 120, settled = true, kind = "sale" },
        { amount = 35, settled = false, kind = "sale" },
        { amount = 10, settled = true, kind = "refund" }
    ]

    revenue = payments
        .filt(e.settled && e.kind == "sale")
        .agg(
            += e.amount
        )

    ret WEB.json({
        user = req.ctx.userId,
        account = req.params.id,
        revenue = revenue
    })
)

This route-first shape is why main() is anti-hoisted: the entry point can read like a table of contents, while middleware and handlers stay below it as ordinary top-level functions. Other functions are not anti-hoisted as a language feature.

Types

Caster has a small set of primitive and runtime-backed types. A named fixed MAP also becomes a type that can be used in function signatures and arrays.

Type Purpose Example
INT 64-bit integer. INT count = 0
FLT Floating-point number. FLT ratio = 1.5
BOL Boolean value. BOL ready = true
STR String value. STR name = "Ada"
NUL No-value return type and null sentinel. FN run() -> NUL
ARR[T] Typed array. ARR[INT] nums = []
MAP Name Named fixed-field value and type. MAP User = { ... }
MAP[K, V] Runtime-key map. MAP[STR, INT] Scores = {}
DYN Runtime-shaped JSON or open data. data = JSON(body)
REF[T] Borrowed reference to a value. REF[Counter] counter
TSK[T] Typed task handle. TSK[REQ.Res]
FN[A -> B] Single-argument function value. FN[WEB.Req -> WEB.Res]

Values And Local Inference

Inside a function, local variables created from literals and ordinary expressions infer their type. The type remains static; inference only removes repetition. 

FN describe(INT count) -> STR = (
    label = "items"
    doubled = count * 2
    ret STR(doubled) + " " + label
)

Call-derived locals

A new local variable initialized directly from a user-written function call declares that function's return type. Collection methods, casts, builtins, native adapters, and ordinary local calculations infer. 

FN example() -> INT = (
    ARR[Order] orders = makeOrders()
    active = orders.filt(e.active)
    count = active.len()
    ret count
)

Structural values

Anonymous MAP-shaped values may conform at a typed argument, return, or declaration boundary when their fields match the target type. 

MAP User = {
    STR name = ""
    INT age = 0
}

FN makeUser() -> User = (
    ret {
        name = "Alan"
        age = 32
    }
)

Functions

Functions are Caster's main typed boundary. Parameters and return values are always explicit, including -> NUL . The body stays lighter by inferring ordinary local work.

Declaring and returning

FN sum(INT a, INT b) -> INT = (
    ret a + b
)

FN printTotal(INT a, INT b) -> NUL = (
    total = a + b
    log(total)
)

Nested helpers

A function may declare a private helper inside its body. Nested helpers are organizational tools, not full closures: outer state is passed explicitly, usually with REF when the helper must mutate it. Nested helpers can return ordinary values, but the helper function itself cannot escape the owning function. 

FN buildScore() -> INT = (
    total = 1
    values = [1]

    FN add(REF[INT] total, REF[ARR[INT]] values, INT value) -> INT = (
        *total += value
        *values.add(value)
        ret total
    )

    INT latest = add(REF total, REF values, 2)
    latest = add(REF total, REF values, 3)

    ret latest
)

MAPs

Caster uses MAP for both fixed-field data and runtime-key maps. A fixed MAP is a named value shape that lowers like a C struct. A dynamic MAP[K, V] is a hashmap whose keys are runtime data.

Fixed MAPs

MAP User = {
    STR name = ""
    INT age = 0
}

Nested declarations keep a one-off shape readable when the nested data belongs to that MAP. Pull the nested shape into its own named MAP when it is reused across function boundaries. 

MAP SalesRegion = {
    STR name = "",
    FLT currencyRate = 1.0,
    ARR[MAP] stores = [{
        STR name = "",
        FLT taxRate = 1.0,
        ARR[MAP] sales = [{
            FLT amount = 0.0,
            BOL completed = false
        }]
    }]
}

Updating MAPs

.upd({...}) creates an updated value. Assign it back when replacing a local value, or assign the updated nested value through a REF when the caller should see the change. Nested MAPs keep their shape, so deep updates stay readable without turning the data into an untyped object. 

MAP Profile = {
    STR name = "",
    MAP settings = {
        STR theme = "light",
        BOL email = true
    }
}

FN updateProfile(Profile profile) -> Profile = (
    ret profile.upd({
        name = "Ada",
        settings = profile.settings.upd({
            theme = "dark"
        })
    })
)

FN muteEmail(REF[Profile] profile) -> NUL = (
    *profile.settings = profile.settings.upd({
        email = false
    })
)

Dynamic MAPs

Dynamic MAP entries use quoted literal keys, or [expression] when the key is computed at runtime. 

MAP[STR, INT] Scores = {
    "math" = 1,
    "science" = 2
}

Default Values

MAP literals can be returned directly into a typed function boundary. Empty nested maps and arrays use the declared defaults from the receiving shape, while explicit fields override the values you care about. 

MAP Student = {
    STR name = "Guest"
    Scores scores = {}
}

FN namedStudent() -> Student = (
    ret {
        name = "Ada",
        scores = {
            "math" = 95
        }
    }
)

Use .upd({...}) when you want to preserve an existing runtime value and override only part of it.

Typed Constructors

TypeName(value) is a fixed MAP conformance boundary. It checks that an anonymous record or open value fits the named shape, fills omitted fields from defaults, rejects unknown fields, and returns the typed value. 

FN typedUser() -> User = (
    User user = User({
        name = "Ada"
    })

    ret user // age uses the User default
)

Arrays

Arrays use ARR[T] in type positions and bracket literals in value positions. Empty literals resolve from a typed declaration, argument, or return boundary. Methods return new arrays unless the call is prefixed with the mutation operator. 

ARR[INT] SeedNums = [1, 2, 3]
ARR[STR] SeedNames = ["Ada", "Grace", "Lin"]
ARR[BOL] Row = []
ARR[Row] Grid = []

FN arrayTour() -> NUL = (
    nums = SeedNums
    names = SeedNames

    nums = nums.add(4)         // [1, 2, 3, 4]
    *nums.add([5, 6])          // nums is now [1, 2, 3, 4, 5, 6]

    combined = nums + [7, 8]   // [1, 2, 3, 4, 5, 6, 7, 8]
    doubled = nums.upd(e * 2)  // [2, 4, 6, 8, 10, 12]
    large = nums.filt(e >= 4)  // [4, 5, 6]
    total = nums.agg( += e )   // 21
    hasLarge = nums.any(e > 5) // true
    sorted = nums.sort(e DESC) // [6, 5, 4, 3, 2, 1]

    byLength = names.group(e.len())
    randomGrid = [10x10].fill(RAND(1) > 0.7)
    volume = [10x10x(3 * 2)].fill(0)
)

fill(value) evaluates the fill expression for each cell. Constants produce repeated values, while calls such as RAND(1) produce fresh values. Each dimension expression is evaluated once and must produce a nonnegative INT .

Strings

Strings are ordinary values with concatenation, indexing, comparison, and runtime helpers for trimming, case conversion, replacement, splitting, and joining. text.ARR() exposes a string as a sequence of one-character strings; parts.STR(delimiter) joins an array back into a string. 

name = "Alan Turing"
letters = name.ARR()  // ["A", "l", "a", ...]
parts = name.ARR(" ") // ["Alan", "Turing"]
joined = parts.STR("-") // "Alan-Turing"

Immediate string loops may lower directly without allocating a separate array. 

loop([name.ARR()])(
    log(e) // "A", "l", "a", ...
)

Universal Dot Access

Dot access is Caster's shared syntax for fixed fields, array indexes, runtime map keys, and DYN traversal. A simple step uses .name or .0 . A computed step uses .[expression] . 

user.name              // fixed MAP field
nums.0                 // literal array index
nums.i                 // simple local index
nums.[i + 1]           // computed index
grid.[row - 1].[col]   // two computed steps

Dynamic keys

On a runtime-key map, dot access uses an in-scope string local when one exists; otherwise the dot name is treated as a literal string key. Use computed access for larger key expressions. 

scores = {
    "math" = 3,
    "science" = 5
}

subject = "science"
score = scores.subject       // 5
computed = scores.[subject]  // 5

DYN traversal

Dot access on DYN remains dynamic until a typed boundary or else fallback extracts a concrete value. 

name = data.owner.name else "Guest"
count = data.items.len else 0

Operators

Operator Meaning
+ , - , * , / , % Numeric arithmetic where supported.
== , != , < , <= , > , >= Comparison.
&& , || , ! Boolean logic.
+= Numeric addition or string append assignment.
else Fallback value and safe DYN extraction. 
count += 1
name += "er"
label = data.label else "unknown"

Arrays and MAPs use named methods for structural mutation. This avoids making += mean append, extend, merge, and overwrite depending on the receiver.

IS Operator

IS establishes one comparison subject and reuses it across the following clauses. A bare clause defaults to equality; a clause beginning with another comparison operator uses that operator. 

if(item.kind IS "LPAREN" || "LBRACE" || "LBRACKET")(
    delta = 1
)

if(res.status IS >= 200 && < 300)(
    ret true
)

if(row IS < 0 || >= len(grid))(
    ret false
)

Canonical style omits an explicit == after IS . Keep ordinary property predicates explicit rather than using IS as a general implicit-receiver scope.

Control Flow

Caster uses parenthesized blocks for functions and control flow. Use if , elx , and else for branching. 

FN classify(INT value) -> STR = (
    if(value < 0)(
        ret "negative"
    ) elx(value == 0)(
        ret "zero"
    ) else(
        ret "positive"
    )
)

A postfix conditional is useful when one expression contributes only under a condition. In ordinary value selection, include else when both outcomes are required. 

label = "alive" if(cell) else "dead"
fee = store.processingFee if(store.active) else 0

Loops

Loops expose a small set of inferred helper values. Counted loops provide i . Array loops provide i and e . Map loops provide e for the key and val for the value.

Counted, range, and condition loops

loop(3)(
    log(i) // 0, 1, 2
)

loop(0, 5)(
    log(e) // 0, 1, 2, 3, 4
)

loop(0, 10, 2)(
    log(e) // 0, 2, 4, 6, 8
)

count = 0
loop(count < 3)(
    log(count) // 0, 1, 2
    count += 1
)

Array and map loops

nums = [10, 20, 30]
loop([nums])(
    log(i, e) // 0 10, 1 20, 2 30
)

scores = {
    "math" = 1,
    "science" = 2
}
loop({scores})(
    log(e, val) // "math" 1, "science" 2
)

Nested loops

The loop source is evaluated before the new loop's e and i bindings are introduced. That lets a nested loop use the outer element as its source, while the complete outer collection still names the outer frame. 

grid = [
    [true, false],
    [false, true]
]

loop([grid])(
    loop([e])(
        rowIndex = grid.i
        colIndex = i
        cell = e
    )
)

Use as only when a middle loop frame must remain addressable inside a third loop. 

loop([cube])(
    loop([e] as rows)(
        loop([rows.e])(
            use(cube.i, rows.i, i, e)
        )
    )
)

Collections

Collection methods are expressions by default. filt keeps elements, upd transforms them, and agg combines numeric or string contributions. Prefix supported methods with * when the receiver should be mutated in place.

Filter and update

nums = [1, 2, 3, 12]

doubled = nums.upd(e * 2) // [2, 4, 6, 24]
big = nums.filt(e > 2)    // [3, 12]

*nums.upd(e * 2)
*nums.filt(e > 4)        // nums is now [6, 24]

Aggregation

agg makes the combining operator explicit. The expression is evaluated once per element and its result is combined into the accumulator. Additive aggregation uses 0 for an empty collection. A conditional contribution may omit else ; a false condition contributes nothing. 

total = nums.agg(
    += e
) // 18

weighted = nums.agg(
    += e * 2
) // 36

countBig = nums.agg(
    += 1 if(e > 10)
) // 1

Nested collection context

Inside a collection expression, a leading dot reads from the nearest current element. Named collection paths also infer a singular alias, which remains available when a deeper operation shadows the nearest element. 

netRevenue = regions.agg(
    += .stores
        .filt(.active)
        .agg(
            += .sales
                .filt(.completed && !.refunded)
                .agg(
                    += (.amount - .discount)
                        * store.taxRate
                        * region.currencyRate
                )
            - (
                store.processingFee
                    if(store.active)
                    else 0
            )
        )
    - region.operatingCost
)

Read the example from the inside out: each qualifying sale contributes a value, stores combine their sales and subtract a store-level fee, and regions combine their stores and subtract a region-level cost.

Boolean helpers

activeRegions = regions.filt(
    .stores.any(.active)
)

any short-circuits when one element matches. The same test can be written as .stores.filt(.active).len() > 0 , but any states the intent directly.

Tap

tap(name)(...) runs a checkpoint block and then returns the original receiver unchanged. It is useful for logging, assertions, or assigning an already-declared outer variable without breaking a chain. A tap is always followed by another collection method; it is not a standalone endpoint. 

totalRevenue = 0

report = orders
    .filt(e.status IS "PAID" || "SHIPPED")
    .tap(rows)(
        totalRevenue = rows.agg(
            += e.revenue
        )
    )
    .filt(e.revenue > 0)

Group

group(keyExpression) creates group rows. Each group exposes key for the grouped value and val for the matching element array. The inferred alias comes from the key expression when that is readable, so orders.group(e.region) exposes region.key and region.val inside the next collection method. 

report = orders
    .group(e.region)
    .upd({
        region = region.key,
        count = region.val.len(),
        revenue = region.val.agg(
            += e.revenue
        )
    })
    .ARR()

Typed aggregate

MAP Payload = {
    ARR[MAP] regions = [{
        STR name = "",
        FLT currencyRate = 1.0,
        FLT operatingCost = 0.0,
        ARR[MAP] stores = [{
            STR name = "",
            BOL active = false,
            FLT taxRate = 1.0,
            FLT processingFee = 0.0,
            ARR[MAP] sales = [{
                FLT amount = 0.0,
                FLT discount = 0.0,
                BOL completed = false,
                BOL refunded = false
            }]
        }]
    }]
}

FN decodePayload(DYN data) -> Payload = (
    ret data
)

FN netRevenue(DYN data) -> FLT = (
    Payload payload = decodePayload(data)

    netRevenue = payload.regions.agg(
        += .stores
            .filt(.active)
            .agg(
                += .sales
                    .filt(.completed && !.refunded)
                    .agg(
                        += (.amount - .discount)
                            * store.taxRate
                            * region.currencyRate
                    )
                - store.processingFee
            )
        - region.operatingCost
    )

    ret netRevenue
)

The expression reads as:

For every region:
    add every active store's result
    subtract the region's operating cost

For every active store:
    add every completed, non-refunded sale's contribution
    subtract the store's processing fee

For every qualifying sale:
    add:
        (amount - discount)
        × store tax rate
        × region currency rate

Language Tour: Conway

This larger example combines named types, typed functions, dot access, nested loops, aggregation, and value-oriented grid updates. It appears here after the individual features so each part has already been introduced. 

ARR[BOL] Row = []

ARR[Row] Grid = []

MAP RenderOpts = {
    STR alive = "#",
    STR dead = "."
}

MAP Offset = {
    INT row = 0,
    INT col = 0
}

ARR[Offset] Offsets = [
    { row = -1, col = -1 },
    { row = -1, col =  0 },
    { row = -1, col =  1 },
    { row =  0, col = -1 },
    { row =  0, col =  1 },
    { row =  1, col = -1 },
    { row =  1, col =  0 },
    { row =  1, col =  1 }
]

FN main() -> NUL = (
    Grid grid = randomGrid()
    opts = {
        alive = "#",
        dead = "."
    }
    generation = 0

    loop(true)(
        log("generation " + STR(generation))
        printGrid(grid, opts)
        log("alive " + STR(countAlive(grid)))
        log("")

        grid = step(grid)
        generation += 1
    )
)

FN randomGrid() -> Grid = (
    ret [10x10].fill(RAND(1) > 0.68)
)

FN aliveAt(Grid grid, INT row, INT col) -> BOL = (
    if(row IS < 0 || >= grid.len())(
        ret false
    )

    if(col IS < 0 || >= grid.row.len())(
        ret false
    )

    ret grid.row.col
)

FN neighborCount(Grid grid, INT row, INT col) -> INT = (
    ret Offsets.agg(
        += 1 if(aliveAt(grid, row + e.row, col + e.col))
    )
)

FN nextCell(Grid grid, INT row, INT col) -> BOL = (
    BOL alive = aliveAt(grid, row, col)
    INT neighbors = neighborCount(grid, row, col)

    if(alive)(
        ret neighbors IS 2 || 3
    )

    ret neighbors == 3
)

FN step(Grid grid) -> Grid = (
    next = []

    loop([grid])(
        row = []

        loop([e])(
            *row.add(nextCell(grid, grid.i, i))
        )

        *next.add(row)
    )

    ret next
)

FN renderCell(BOL alive, RenderOpts opts) -> STR = (
    if(alive)(
        ret opts.alive
    )

    ret opts.dead
)

FN renderRow(Row row, RenderOpts opts) -> STR = (
    ret row.agg(
        += renderCell(e, opts)
    )
)

FN printGrid(Grid grid, RenderOpts opts) -> NUL = (
    loop([grid])(
        log(renderRow(e, opts))
    )
)

FN countAlive(Grid grid) -> INT = (
    ret grid.agg(
        += .agg(
            += 1 if(e)
        )
    )
)

REF And Mutation

        

          The
           * 
          token marks an in-place write. It can mutate an owned local value or a
          value reached through
           REF 
          .
           REF 
          is needed only when a function must borrow storage owned by its
          caller.
        

        
nums = [1, 2, 3]
*nums.add(4) // mutates an owned local

FN addOne(REF[ARR[INT]] nums) -> NUL = (
    *nums.add(1) // mutates the caller's array
)

addOne(REF nums)

        

          Reads through a reference are automatic. The mutation marker stays at
          the write site so the line that changes storage remains visible. A
          source-written
           * 
          inside a condition, call argument, or return expression is rejected;
          use the reference name directly for reads.
        

        
MAP Counter = {
    INT value = 0
}

FN bump(REF[Counter] counter) -> NUL = (
    *counter.value += 1
)

        

          A function-local borrow may be passed downward to helpers, but it
          cannot be returned or stored somewhere that may outlive its owner.
        

      

      

        
JSON And DYN

        

           DYN 
          represents runtime-shaped data, usually produced by
           JSON(str) 
          . Dot and index access on DYN remain dynamic. Decode at a typed
          boundary before the main calculation, or use
           else 
          when a local fallback is sufficient.
        

        
MAP Account = {
    INT id = 0,
    FLT balance = 0.0,
    BOL active = false,

    MAP owner = {
        STR name = "Guest",
        STR email = ""
    }
}

FN decodeAccount(DYN data) -> Account = (
    ret {
        id = data.id else 0,
        balance = data.balance else 0.0,
        active = data.active else false,
        owner = {
            name = data.owner.name else "Guest",
            email = data.owner.email else ""
        }
    }
)

        

          Typed decode currently supports fixed MAPs, dynamic
           MAP[STR, T] 
          , nested MAPs, arrays, FLT values, JSON
           null 
          as NUL-compatible data, and nested DYN access with safe fallback.
        

      

      

        
SQL

        

          The native
           SQL 
          module wraps SQLite for small embedded databases. Statements use
          positional parameters, execution results are typed, and query rows
          return
           DYN 
          so callers can decode into their own MAP shapes at the boundary.
        

        
use SQL

MAP User = {
    INT id = 0,
    STR name = "",
    BOL active = false
}

FN decodeUsers(DYN data) -> ARR[User] = (
    ret data
)

FN main() -> NUL = (
    db = SQL.open("app.db")

    SQL.exec(REF db, `
        CREATE TABLE IF NOT EXISTS users (
            id INTEGER PRIMARY KEY,
            name TEXT NOT NULL,
            active INTEGER NOT NULL
        )
    `, [])

    inserted = SQL.exec(REF db, `
        INSERT INTO users (name, active)
        VALUES (?, ?)
    `, ["Ada", true])

    log("inserted " + STR(inserted.rows)) // inserted 1

    rows = SQL.query(REF db, `
        SELECT id, name, active
        FROM users
        WHERE active = ?
        ORDER BY name
    `, [true])

    ARR[User] users = decodeUsers(rows)

    loop([users])(
        log(e.id, e.name) // 1 Ada
    )

    SQL.close(REF db)
)

      

      

        
Tasks And Hold

        

           TSK[T] 
          is a typed wrapper around a generic task handle. Requests create hot
          tasks.
           hold task 
          waits for one task.
           hold [taskA, taskB] 
          waits for all tasks and returns results in input order.
        

        
TSK[REQ.Res] userTask = REQ.get(base + "/users/101")
TSK[REQ.Res] postsTask = REQ.get(base + "/users/101/posts")

ARR[REQ.Res] results = hold [userTask, postsTask]

user = results.0
posts = results.1

      

      

        
REQ

        

           REQ 
          is the lower-level HTTP layer. Client requests return
           TSK[REQ.Res] 
          . Hosting starts a blocking H2O-backed HTTP server, with
           REQ.hostTLS 
          available for certificate-backed HTTPS. Low-level
           REQ.ws 
          and
           REQ.wsTLS 
          expose WebSocket servers when H2O is built with wslay.
        

        
TSK[REQ.Res] task = REQ.get("http://localhost:8000/")
REQ.Res res = hold task

TSK[REQ.Res] postTask = REQ.post("http://localhost:8000/echo", `{"name":"Alan"}`)
REQ.Res postRes = hold postTask

        
FN status(REQ.Req req) -> REQ.Res = (
    ret {
        status = 200,
        body = "ok",
        headers = {}
    }
)

FN main() -> NUL = (
    REQ.host(8080, status)
)

        
FN echoSocket(STR message) -> STR = (
    ret message
)

FN main() -> NUL = (
    REQ.ws(8081, echoSocket)
)

           use NAME

Advanced

        
Memory

        

          Caster is value-oriented until a function explicitly asks for a
          reference. The practical pattern is to let an outer function own the
          state, then define nested helpers that receive
           REF 
          parameters for the exact values they mutate. The helper cannot escape
          the outer function, and every mutation is visible at the call site
          through
           REF value 
          and the
           * 
          mutation operator.
        

        
MAP Node = {
    INT value = 0,
    REF[Node] next = NUL
}

MAP List = {
    REF[Node] head = NUL,
    REF[Node] tail = NUL,
    INT count = 0
}

FN testNestedAppendAndPrint() -> BOL = (
    list = { head = NUL, tail = NUL, count = 0 }
    first = { value = 0, next = NUL }
    second = { value = 0, next = NUL }
    third = { value = 0, next = NUL }

    FN append(REF[List] list, REF[Node] node, INT value) -> NUL = (
        *node.value = value
        *node.next = NUL

        if(list.head == NUL)(
            *list.head = node
            *list.tail = node
        ) else (
            tail = list.tail
            *tail.next = node
            *list.tail = node
        )

        *list.count += 1
    )

    FN printList(REF[List] list) -> NUL = (
        current = list.head

        loop(current != NUL)(
            log(current.value) // 10, 20, 30
            current = current.next
        )
    )

    append(REF list, REF first, 10)
    append(REF list, REF second, 20)
    append(REF list, REF third, 30)

    printList(REF list)

    ret list.count == 3
        && list.head.value == 10
        && list.head.next.value == 20
        && list.tail.value == 30
)

        
What Owns What

        

          In the linked-list example, the outer function owns
           list 
          ,
           first 
          ,
           second 
          , and
           third 
          . The list stores references to those nodes, but the references do not
          own the nodes. When the function ends, the owned locals are cleaned up
          by the normal function cleanup path.
        

        

           REF[T] 
          appears in type positions: MAP fields and function signatures. It
          means "address to
           T 
          ".
           REF value 
          appears in expression positions and creates a borrowed reference to an
          existing value. It does not transfer ownership and it cannot be
          manually freed.
        

        
Mutation Operator

        

          Reads through a reference are intentionally light:
           list.head 
          reads the current head,
           current.value 
          reads a node value, and
           current.next 
          follows the link. The
           * 
          token marks caller-visible mutation:
           *list.head = node 
          ,
           *tail.next = node 
          , and
           *list.count += 1 
          all mutate through a reference. The mutation operator is
          statement-only, so read expressions use the normal name:
           len(list) 
          instead of
           len(*list) 
          .
        

        
Escape Rule

        

          The safety rule is simple: a stored reference must not outlive the
          value it points at. Borrowing local nodes into a local list is fine
          while the list stays inside the same owning function. Returning that
          reference-shaped list, assigning it to a global, or capturing it in a
          longer-lived task would require the compiler to prove the nodes live
          long enough; otherwise it is rejected.
        

        
FN badReturn() -> REF[Node] = (
    node = { value = 0, next = NUL }

    ret REF node // rejected: node is function-local
)

FN badStore() -> NUL = (
    left = { value = 0, next = NUL }
    right = { value = 0, next = NUL }

    left.next = REF right // rejected if the stored borrow can escape
)

        
Returning Durable Structures

        

          When a linked structure must be returned, prefer a value-owned shape
          rather than REF fields to local nodes. An array-backed list stores
          nodes by value and links by index, so the returned
           List 
          owns the full reachable structure.
        

        
MAP ValueNode = {
    INT value = 0,
    INT next = -1
}

MAP ValueList = {
    INT head = -1,
    ARR[ValueNode] nodes = []
}

        
Automatic Cleanup

        

          Primitive values copy directly. Heap-backed values include
           STR 
          ,
           ARR[T] 
          , dynamic
           MAP[K, V] 
          , open MAP values,
           DYN 
          /JSON objects, and task handles. The compiler tracks owned locals and
          emits cleanup on explicit
           ret 
          , function fallthrough, and hidden script-main fallthrough. Returning
          an owned local transfers that value to the caller; other owned locals
          are cleaned up first.
        

        

          Hosted
           REQ.host 
          /
           WEB 
          requests also have a runtime cleanup boundary. The host drops concrete
          request strings/maps and response strings/maps after dispatch and
          response write. Handler-local values still follow normal compiler
          cleanup rules.
        

        
Manual FREE

        

           FREE(value) 
          and
           FREE value 
          are available for early cleanup when the analyzer can prove the local
          owns heap-backed runtime storage. Borrowed references cannot be freed:
          the owning value is responsible for cleanup.
        

        
FN borrowedRef() -> NUL = (
    value = 10
    ptr = REF value

    FREE ptr // rejected: borrowed reference
)

        
Runtime Allocation Sites

        

           STR 
          carries a data pointer, length, and owned flag in generated C. String
          literals are borrowed/static; concatenation, string transforms, JSON
          parsing, and HTTP body handling create owned strings. Arrays lower to
          structs with
           data 
          ,
           len 
          , and
           cap 
          . Dynamic maps and open objects use
           hashmap.c 
          ; string-keyed map entries own copied keys, and
           MAP[STR, STR] 
          also owns copied string values.
        

        

           JSON(str) 
          creates
           DYN 
          values backed by internal object, array, string, number, boolean, and
          NUL-compatible storage.
           REQ.get/post/put/delete 
          return hot
           TSK[Res] 
          values backed by generic task handles with runtime destroy callbacks.
